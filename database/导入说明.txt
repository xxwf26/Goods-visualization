数据库导入说明
================

本压缩包含周边可视化系统的完整数据库（结构 + 数据），共 20 张表。
导出工具：mysqldump（MariaDB 12.3），字符集 utf8mb4。

【文件】
goods_visualization_full_dump.sql —— 完整数据库导出（含建库、建表、数据）

【导入方式】（任选其一）

方式一：命令行（推荐）
  mysql -u<你的用户名> -p<你的密码> < goods_visualization_full_dump.sql
说明：dump 文件内含 CREATE DATABASE IF NOT EXISTS goods_visualization 和 USE goods_visualization，
      所以直接导入即可，会自动建库 goods_visualization 并写入数据，无需手动建库。
      导入后你的库里会有：goods_visualization 库 + 20 张表 + 全部业务数据。

方式二：Navicat / 其它 GUI 工具
  1. 连接你的 MySQL
  2. 新建查询，把 goods_visualization_full_dump.sql 全部内容粘进去执行
  （或右键库 → 运行 SQL 文件 → 选这个 sql）

【导入后配置】
1. 确保 backend/.env 里的数据库配置指向这个库：
   DB_HOST=127.0.0.1
   DB_PORT=3306（按你的 MySQL 端口改）
   DB_NAME=goods_visualization
   DB_USER=<你的用户名>
   DB_PASSWORD=<你的密码>
2. 重启后端服务。

【默认账号】（导入后可用这些账号登录系统）
  admin / admin123        （管理员）
  其它账号见 sys_user 表，密码同样是加密存储的，可联系原库主人确认。

【注意事项】
- 本 dump 含原库的全部业务数据（灵感、项目、价格、供应商、标签、用户、操作日志等），
  导入后你将得到与原库一致的完整环境。
- 字符集 utf8mb4，支持 emoji 和中文，导入时若遇乱码确认 MySQL 连接字符集为 utf8mb4。
- dump 用了 --single-transaction 导出，是一致性快照。
- 如已有同名库 goods_visualization，导入不会清空已有表（CREATE TABLE IF NOT EXISTS），
  但 INSERT 会插入重复数据。建议导入前先 DROP DATABASE goods_visualization 或换库名。

【版本】
含所有结构变更（v1 ~ v18），包括：
- inspiration_tag / project_tag 标签关联表
- inspiration_candidate / crawl_run 灵感采集表
- play_count 播放量字段、tag 表索引等
无需再单独跑 fix_schema_v*.sql。
