周边采购灵感与历史价格知识库网页需求文档
一、项目背景
目前周边制作采购过程中，历史项目数据、历史报价、供应商信息、工艺参考、外部灵感链接等信息分散在飞书多维表格、聊天记录、小红书收藏、淘宝链接、文件夹图片和个人经验中。
在新项目启动时，采购或需求方经常需要重复查找以下信息：
1. 以往是否做过同类周边；
2. 该品类历史单价、数量、供应商、打样周期、大货周期；
3. 同类工艺过往效果如何；
4. 小红书/淘宝/其他平台近期有哪些可参考的新工艺、新玩法、新包装；
5. 某个IP或某个品类适合哪些物料形式；
6. 当前询价是否明显高于历史水平。
因此希望搭建一个内部网页工具，用于沉淀和查询“历史周边数据 + 外部灵感参考 + 价格/工艺经验”，提升采购前期调研、报价审核和方案发散效率。
二、项目目标
搭建一个可检索、可筛选、可持续更新的网页看板/知识库，实现以下目标：
1. 快速查询历史周边制作记录；
2. 快速查询历史品类价格、数量、供应商、周期等信息；
3. 按品类沉淀外部灵感链接，如小红书、淘宝、1688、站酷、微博等；
4. 支持按“品类 / 工艺 / IP / 供应商 / 单价区间 / 时间”筛选；
5. 支持将内部历史案例与外部参考链接放在同一页面对照查看；
6. 为采购询价、需求讨论、供应商brief、报价审核提供参考依据。
三、使用人群
1. 采购同事：用于历史价格查询、供应商参考、报价审核；
2. 市场/运营/活动同事：用于周边品类发想和外部灵感参考；
3. 设计同事：用于工艺、包装、材质和视觉呈现参考；
4. 管理者：用于查看周边项目投入、供应商分布、品类成本趋势。

用户想做“明信片”，进入系统后搜索“明信片”，可以看到：

7. 供应商；

9. 大货周期；

用户选择“近期明信片工艺参考”，可以看到外部收集链接，例如：
1. 小红书明信片工艺案例；

1. 我们以前做过什么；

3. 同供应商历史报价；
4. 其他供应商同类报价；

八、数据更新方式
方案A：人工维护版
第一阶段建议先做人工维护。
使用方式：
1. 采购或需求方手动新增链接；
2. 填写品类、工艺、IP、适用场景等标签；
3. 上传截图；
4. 填写参考价值说明；
5. 定期从飞书多维表格同步历史项目数据。
优点：
1. 成本低；
2. 开发简单；
3. 数据质量高；
4. 合规风险低。
方案B：半自动维护版
第二阶段可考虑半自动化。
功能：
1. 用户粘贴小红书/淘宝链接；
2. 系统自动读取标题、封面、来源平台；
3. 用户补充标签和参考说明；
4. 系统定期检查链接是否失效。
注意：
小红书和淘宝存在登录、反爬和链接失效问题，不建议第一阶段直接做全自动抓取。
方案C：AI辅助整理版
第三阶段可接入AI能力。
功能：
1. 自动识别外部链接内容；
2. 自动建议品类标签和工艺标签；
3. 自动总结参考价值；
4. 自动匹配内部历史案例；
5. 自动生成“本周周边灵感摘要”。
九、AI功能设想
1. AI自动打标签
用户新增一个链接后，AI根据标题、截图或说明自动推荐：
1. 品类；
2. 工艺；
3. 适用IP；
4. 适用场景；
5. 参考价值。
2. AI总结外部参考
例如用户保存一条小红书明信片链接，AI自动生成：
“该案例主要参考点为珠光纸材质、套组式包装、低饱和粉色系视觉，适合用于闪耀暖暖线下活动赠品或艺术展入场礼。”
3. AI匹配历史案例
当用户查看“明信片工艺参考”时，AI自动推荐：
1. 公司历史明信片项目；
2. 相近工艺；
3. 相近单价；
4. 相近供应商；
5. 需要注意的成本风险。
4. AI生成采购参考
用户输入：
“闪耀暖暖线下活动，5元以内周边，有什么新意？”
AI根据内部价格库和外部灵感库，输出：
1. 推荐品类；
2. 历史单价；
3. 外部参考链接；
4. 适合工艺；
5. 推荐供应商；
6. 风险点。

小红书、淘宝等外部平台链接建议先采用人工录入方式，避免因平台登录、反爬、链接变化导致系统不稳定。后续如确有需要，再评估是否接入半自动解析或AI摘要能力。
